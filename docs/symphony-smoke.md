# Symphony smoke test evidence checklist

This document describes the evidence expected from a successful disposable
run of the experimental GitHub-native Symphony worker against this repository
(see the **Experimental Symphony worker** section in `README.md`). It is
specific to `ttrpg-sim` smoke-test issues and is not a general Symphony guide.

Use this checklist to confirm a smoke run completed correctly before deciding
whether to review or discard its output.

## Expected evidence

- **Issue status transition** — the smoke issue's status label moves from
  `symphony:test-working` to `symphony:test-review` (it starts eligible only
  once it carries both `symphony:test` and `symphony:test-ready`).
- **Bot-owned status comment** — the worker posts at least one comment on the
  issue, authored by the bot/automation identity (not a human), summarizing
  the work performed.
- **Coordination refs** — one or more refs are created under
  `refs/heads/symphony/` in this repository, used by the worker to coordinate
  state across the run.
- **Test-work branch** — a branch matching `symphony/test-work/*` is pushed,
  containing the changes generated for the smoke issue.
- **Generated pull request** — a pull request is opened from the
  `symphony/test-work/*` branch and labeled `symphony:test-generated`.

## Notes

- This checklist only documents disposable smoke-test evidence; it does not
  change calculator behavior or the existing `agent:*` automation.
- Do not merge a pull request generated by a smoke run — review it, confirm
  the evidence above, and then close or discard it per the smoke test's
  purpose.
